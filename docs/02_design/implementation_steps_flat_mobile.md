# 実装ステップ: Mobile-First & Flat Design

目標: モバイルファーストかつフラットなデザインへの完全移行を、段階的に進めます。
特にモバイル時の「横並びリストレイアウト」とPC時の「グリッドレイアウト」の使い分けを重視します。

## Phase 1: 基盤スタイルの刷新 (Variables & Reset)
まず、`style.css` のルート変数を修正し、デザインシステム全体にフラットデザインを適用します。

### 1-1. 変数の再定義 (`:root`)
- [ ] `style.css`: `--shadow-*` 系変数を全て `none` または `0 0 0` に変更し、影を廃止。
- [ ] `style.css`: `--gradient-*` 系変数を削除し、ソリッドカラー (`--color-tech-blue` 等) に置き換え。
- [ ] `style.css`: `--border-radius` 変数を `4px` (または `8px`) に統一し、Pill型を廃止。

### 1-2. リセット & ベーススタイル
- [ ] ボタンや入力フォームのブラウザデフォルト立体表現（`appearance`）を無効化。
- [ ] リンク色やホバー挙動を「色変化」のみ（移動や影なし）に統一。

## Phase 2: コンポーネントのフラット化
各 UI コンポーネントから「立体感」を排除し、境界線ベースのデザインへ変更します。

### 2-1. ボタン (Buttons)
- [ ] `.button`: グラデーション背景を単色に変更。影を削除。
- [ ] `.button`: `border-radius` を `4px` に変更。
- [ ] `.button.outline`: ボーダーの太さを調整。

### 2-2. カード (Cards)
- [ ] `.article-card`: `box-shadow` を削除し、`border: 1px solid var(--color-border-gray)` を適用。
- [ ] `.article-thumbnail`: アスペクト比を維持しつつ、フラットな見た目に。

## Phase 3: レスポンシブレイアウトの最適化 (核心)
PCとモバイルで最適なレイアウトを適用します。

### 3-1. 記事リストのレイアウト分岐
- [ ] **PC (>768px)**: 既存のグリッドレイアウト（3カラム、画像上・テキスト下）を維持。
- [ ] **Mobile (<768px)**:
    - グリッドを解除し、シングルカラム（1カラム）に設定。
    - **[NEW] 横並びリスト化**: Flexbox/Gridを使用し、「左：サムネイル」「右：タイトル・メタ情報」の配置に変更。
    - 縦スクロール量を削減し、一覧性を高める。

### 3-2. ヘッダー & ナビゲーション
- [ ] `.site-header`: 下部ボーダーのみのフラットデザインに変更。
- [ ] ハンバーガーメニュー: アイコンを視認性の高いフラットスタイルに調整。

### 3-3. サイドバー制御
- [ ] Mobile: サイドバー (`#secondary`) を `display: none` またはメインコンテンツ下部へ移動。記事一覧では非表示推奨。

## Phase 4: トップページセクションの整備
`front-page.php` の各セクション表示をワイヤーフレームに合わせます。

- [ ] **Global Trends**: タブ表示または横スクロールUIのスタイル調整。
- [ ] **By Issues**: 課題別アイコンとリストのスタイル調整。

## Phase 5: 検証 (Verification)
- [ ] ブラウザ検証 (Mobile 375px / Desktop 1200px)。
- [ ] モバイルでの記事リストが「横並び」になっているか確認。
- [ ] 全体的に「影」や「グラデーション」が残っていないか確認。
