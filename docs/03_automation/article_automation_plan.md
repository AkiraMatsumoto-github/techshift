# 記事自動生成・投稿システム導入計画

## 1. 目的
LogiShiftのコンテンツ拡充を加速させるため、AI（Gemini）を活用した記事の自動生成およびWordPressへの自動投稿フローを構築する。
「物流担当者と経営層のための課題解決メディア」というコンセプトを守りつつ、高品質な記事を効率的に量産する仕組みを目指す。

## 2. システム構成案

### アーキテクチャ
- **実行環境**: ローカル（Mac）または クラウド（Cloud Functions等）
- **開発言語**: Python 3.x
- **AIモデル**: Google Gemini Pro (via Google Generative AI SDK)
- **CMS連携**: WordPress REST API
- **画像生成**: Vertex AI Imagen または DALL-E 3 (オプション)

### 処理フロー概要
1.  **Input**: キーワード または ニュース記事URL
2.  **Process (AI)**:
    - 構成案作成（H2/H3）
    - 本文執筆（Markdown/HTML）
    - メタデータ生成（タイトル、抜粋、タグ）
3.  **Output**: WordPressへ投稿（ステータス：下書き）

## 3. 実装フェーズ

### Phase 1: キーワード指定型・単発生成ツール (MVP)
**まずはここから着手します。**
*   **機能**:
    *   Pythonスクリプトを実行し、引数で「キーワード」を渡す。
    *   GeminiがSEOを意識した構成案を作成し、記事を執筆。
    *   WordPress REST API経由で「下書き」として保存。
*   **メリット**:
    *   実装が容易（数日レベル）。
    *   人間が最終チェック（リライト）を行う運用フローを確立しやすい。

### Phase 2: バッチ処理・画像生成連携
*   **機能**:
    *   CSVファイルからキーワードリストを一括読み込み。
    *   記事内容に合わせてアイキャッチ画像を自動生成し、メディアライブラリにアップロードして紐付け。
    *   連続処理で複数の下書きを一気に作成。

### Phase 3: トレンド検知・完全自動化
*   **機能**:
    *   Google News RSSなどを定期巡回し、「物流」「2024年問題」などのトピックを検知。
    *   最新ニュースの要約＋考察記事を自動生成。
    *   Slack等に通知し、承認ボタンで公開（Human-in-the-loop）。

## 4. 技術スタック詳細

### 必要なPythonライブラリ
*   `google-generativeai`: Gemini API操作
*   `requests`: WordPress REST API操作
*   `python-dotenv`: 環境変数管理（APIキーなど）
*   `pandas`: キーワード管理（Phase 2以降）

### WordPress側の準備
*   **Application Passwords**: REST API認証用。
*   **REST API**: 標準で有効化されている `/wp-json/wp/v2/posts` を使用。

## 5. 品質管理とリスク対策 (Quality Control)

### ハルシネーション（嘘情報）対策
*   **運用ルール**: 必ずステータスを「**Draft（下書き）**」で投稿する。
*   **プロンプト設計**: 「架空の事例を作らない」「統計データは出典を明記する」等の制約を厳しく設定。

### SEO品質の担保
*   **構成の型化**: 「導入→課題提起→解決策→まとめ」というLogiShiftの勝ちパターンをプロンプトに組み込む。
*   **文字数制御**: 薄いコンテンツにならないよう、セクションごとの文字数目安を指示。

## 6. 次のアクションプラン
1.  **環境構築**: Python環境の準備。
2.  **接続テスト**: WordPress REST APIへの投稿テスト。
3.  **プロトタイプ作成**: キーワードを渡して記事本文（テキストのみ）を生成・投稿するスクリプトの実装。
