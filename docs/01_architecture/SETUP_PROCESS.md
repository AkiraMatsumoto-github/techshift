# WordPress 環境構築プロセス for LogiShift

このドキュメントは、LogiShiftのWordPressサイトを立ち上げ、カスタムテーマ開発を開始できる状態にするまでの手順を定義します。

## フェーズ1: 環境準備編（アカウント開設など）

Webサイトの「土地」と「住所」を用意するフェーズです。

### 1. レンタルサーバーの契約

- **目的:** Webサイトのデータを保管し、インターネットに公開するためのサーバーを借ります。
- **アクション:** `site_architecture.md` に記載のあるようなレンタルサーバー（例: [Xserver](https://www.xserver.ne.jp/), [ConoHa WING](https://www.conoha.jp/wing/)）と契約します。
- **ポイント:** 多くの場合、無料お試し期間や初期費用無料キャンペーンがあります。まずは試してみるのがお勧めです。

### 2. 独自ドメインの取得

- **目的:** `logishift.com` のような、サイトの永続的な「住所」となるドメインを取得します。
- **アクション:** サイト名に基づいたドメイン名（例: `logishift.jp`, `logishift.media` など）を決定し、取得します。
- **ポイント:** レンタルサーバーの契約と同時に申し込むと、ドメインが無料になる特典があることが多いです。

### 3. サーバーとドメインの紐付け

- **目的:** 取得したドメインにアクセスした際に、契約したサーバーが表示されるように設定します。
- **アクション:** レンタルサーバーとドメインを同じ会社で契約した場合、この設定は自動的に行われることがほとんどです。別々の会社で契約した場合は、ドメイン管理画面でDNS（ネームサーバー）の設定変更が必要です。

## フェーズ2: WordPressインストール編

用意した環境に、WordPressという「家」を建てるフェーズです。

### 1. SSL設定（HTTPS化）

- **目的:** サイトの通信を暗号化し、セキュリティを高めます。GoogleのSEO評価にも影響する重要な設定です。
- **アクション:** レンタルサーバーの管理画面から、「無料独自SSL」などのメニューを選択し、取得したドメインに設定します。通常、数クリックで完了します。

### 2. WordPress 簡単インストール

- **目的:** サーバーにWordPress本体をインストールします。
- **アクション:** レンタルサーバーの管理画面にある「WordPress簡単インストール」機能を利用します。インストール時に以下の情報を入力する必要があります。
    - **サイトのタイトル:** `LogiShift`
    - **ユーザー名:** （WordPress管理画面へのログイン用。推測されにくいもの）
    - **パスワード:** （強力なパスワード）
    - **メールアドレス:** （WordPressからの通知用）

### 3. ログイン確認

- **目的:** WordPressが正しくインストールされ、管理画面にアクセスできることを確認します。
- **アクション:** `http://あなたのドメイン/wp-admin/` にアクセスし、設定したユーザー名とパスワードでログインできることを確認します。

## フェーズ3: WordPress 初期設定編

インストールしたWordPressを、メディアサイトとして最適化するフェーズです。

### 1. パーマリンク設定の変更

- **目的:** 記事ページのURLを、SEOに有利で分かりやすい形式（例: `.../what-is-wms/`）にします。
- **アクション:** WordPress管理画面の `設定 > パーマリンク設定` を開き、共通設定を「**投稿名**」に変更して保存します。

### 2. 不要なプラグイン・テーマの削除

- **目的:** 初期状態でインストールされている不要な要素を削除し、サイトを軽量かつ安全に保ちます。
- **アクション:**
    - `プラグイン > インストール済みプラグイン` を開き、「Hello Dolly」などを削除します。
    - `外観 > テーマ` を開き、今後使わないデフォルトテーマ（例: "Twenty Twenty-Three"など、一つは残す）を削除します。

### 3. 一般設定の確認

- **目的:** サイトの基本的な情報が正しく設定されているか確認します。
- **アクション:** `設定 > 一般` を開き、「サイトの言語」が日本語に、「タイムゾーン」が `東京` になっていることを確認します。

---

**このプロセスが完了すると、WordPressサイトが完全に準備され、いよいよ私たちでカスタムテーマを開発するステップに進むことができます。**
